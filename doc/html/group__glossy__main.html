Start Glossy and stall all other application tasks.

Parameters
data_A pointer to the flooding data. 
              At the initiator, Glossy reads from the given memory
              location data provided by the application.

              At a receiver, Glossy writes to the given memory
              location data for the application.
data_len_Length of the flooding data, in bytes.
initiator_Not zero if the node is the initiator, zero if it is a receiver.
dst_Node id of the destination node.
slack_Slack.
tx_max_Maximum number of transmissions (N).
header_Application-specific header (value between 0x0 and 0xf).
t_stop_Time instant at which Glossy must stop, in case it is still running.
cb_Callback function, called when Glossy terminates its execution.
rtimer_First argument of the callback function.
ptr_Second argument of the callback function.
